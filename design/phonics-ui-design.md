# 自然拼读模块 - UI 设计稿

## 1. 整体架构

### 1.1 底部 Tab 导航
现有布局：[首页] → [我的]

新增后布局：
```
┌─────────────────────────────────────────────────────────┐
│                                                          │
│                      内容区域                            │
│                                                          │
├─────────────────────────────────────────────────────────┤
│           [🏠首页]    [📖拼读]    [👤我的]              │
│             首页        拼读        我的                │
└─────────────────────────────────────────────────────────┘
```

> 入口位置：在"首页"和"我的"之间新增"拼读"Tab，保持首页和我的布局不变

## 5. 状态设计

### 5.1 字母状态

**未学习状态**
```
┌─────────┐
│         │
│    A    │   白色背景
│         │   灰色边框 1px
│         │   灰色字母
└─────────┘
```

**已学习状态**
```
┌─────────┐
│  ╭───╮ │
│  │ ✓ │ │   绿色渐变背景
│  ╰───╯ │   白色字母
│    A    │   绿色勾选图标
└─────────┘
```

### 5.2 按钮状态

| 状态 | 样式 |
|------|------|
| 默认 | 蓝色渐变背景，白色文字 |
| 按下 | 缩小95%，加深颜色 |
| 禁用 | 灰色背景，不可点击 |

- 字母网格：4列
- 字母方块：72x72px
- 间距：8px

### 6.2 平板端 (768px+)
- 字母网格：6列

## 9. 与现有页面的关系

### 9.1 首页布局（保持不变）
- 保持现有的课程卡片入口
- 保持现有的打卡入口
- 保持现有的成就入口

### 9.2 我的页面布局（保持不变）
- 保持现有的用户信息展示
- 保持现有的设置入口
- 保持现有的数据统计

### 9.3 新增拼读页面
- 仅在 Tab 导航中新增"拼读"入口
- 拼读页面采用与首页一致的 UI 风格
- 拼读页面内部导航采用深蓝渐变导航栏
